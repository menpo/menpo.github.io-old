class menpo.io.mesh.base.MeshImporter(filepath, texture=True)[source]¶

Bases: menpo.io.base.Importer

Abstract base class for importing meshes. Searches in the directory specified by filepath for landmarks and textures with the same basename as the mesh. If found, they are automatically attached. If a texture is found then a menpo.shape.mesh.textured.TexturedTriMesh is built, else a menpo.shape.mesh.base.Trimesh is built. Note that this behavior can be overridden if desired.

class menpo.io.mesh.base.WRLImporter(filepath)[source]¶

Bases: menpo.io.mesh.base.MeshImporter

Allows importing VRML meshes. Uses a fork of PyVRML97 to do (hopefully) more robust parsing of VRML files. It should be noted that, unfortunately, this is a lot slower than the C++-based assimp importer.

VRML allows non-triangular polygons, whilst our importer pipeline doesn’t. Therefore, any non-triangular polygons are dropped. VRML also allows separate texture coordinate indices, which we do not support. To have a better formed mesh, try exporting the WRL as OBJ from Meshlab.

Parameters:

filepath : string

Absolute filepath of the mesh.

Interface to meshlabserver to perform prepossessing on meshes before import. Returns a path to the result of the meshlabserver call, ready for import as usual. Requires Meshlab to be installed.

Parameters:

Absolute filepath to the mesh

If specified this script will be run on the input mesh.

Default: None

The output directory for the processed mesh.

Default: The users tmp directory.

The output filetype desired from meshlabserver. Takes the form of an extension, eg obj.
