Wed 30 July 2014 [ Patrick Snape ] | | |

Why develop on Windows?

The Menpo team is dedicated to providing conda packages for all platforms. By far the biggest hurdle towards this goal was successfully building our package on Windows. There is one major reason for this: Visual Studio 2008 1 2. Visual Studio 2008 (VS2008) is a requirement for building Python extensions on Windows. This is due to the fact that compiled extensions link against a specific C++ redistributable.
